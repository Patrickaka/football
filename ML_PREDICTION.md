# 福彩 3D ML 预测功能说明

## 功能概述

已在 index.html 中添加了 **"🤖 运行 ML 预测"** 按钮，可以通过网页界面直接执行机器学习预测。

## 使用方法

1. **启动服务器**
   ```bash
   python server.py
   ```

2. **打开浏览器**
   - 访问 http://localhost:9000
   - 切换到 "🎲 福彩3D" 标签页

3. **运行 ML 预测**
   - 点击页面底部的 **"🤖 运行 ML 预测"** 按钮
   - 等待 30-60 秒（训练模型需要时间）
   - 查看预测结果

## ML 预测结果

ML 预测结果包含以下信息：

### 1. 模型信息
- **模型类型**: 简易随机森林（30 棵树）
- **训练样本**: 总样本数（正例 + 负例）
- **特征维度**: 36 维特征向量

### 2. 直选推荐
- **推荐注数**: 15 注（可配置）
- **排序方式**: 按预测概率降序
- **Top 3 重点推荐**: 单独醒目显示

### 3. 特征列表
- 显示前 10 个最重要的特征
- 包括：热度、遗漏、马尔可夫、组合特征等

## 技术实现

### 前端 (index.html)
- 添加了 `runMLPrediction()` 函数
- 调用 `/api/3d-ml` 接口
- 使用 `renderMLResult()` 渲染结果

### 后端 (server.py)
- 添加了 `_lottery_3d_ml_payload()` 方法
- 导入 `lottery3d_ml` 模块
- 格式化返回结果

### ML 模型 (lottery3d_ml.py)
- 36 维特征工程
- 简易随机森林（纯 Python 实现）
- 时序验证（前 80% 训练，后 20% 验证）

## 性能说明

- **训练时间**: 约 **15-30 秒**（优化后）
  - 树的数量：20 棵（原 30 棵）
  - 树深度：2 层（原 3 层）
  - 负例采样：30 个/期（原 50 个）
  - 滑动窗口：60 期（减少计算量）
- **内存占用**: 约 50-100 MB
- **CPU 使用**: 单核 100%（训练期间）

## 优化策略

为了将预测时间控制在 30 秒以内，我们采用了以下优化：

1. **减少树的数量**: 从 30 棵减少到 20 棵
2. **降低树深度**: 从 3 层减少到 2 层
3. **减少负例采样**: 从 50 个减少到 30 个
4. **滑动窗口**: 只使用最近 60 期数据构建特征
5. **减少最小历史期数**: 从 50 期减少到 30 期

## 优化建议

如果需要更快的训练速度，可以进一步调整参数：

1. **减少树的数量**
   ```python
   N_TREES = 15  # 改为 15 棵
   ```

2. **减少负例采样数**
   ```python
   NEGATIVE_SAMPLES_PER_PERIOD = 20  # 改为 20 个
   ```

3. **减小滑动窗口**
   ```python
   window_size = 40  # 只使用最近 40 期
   ```

**注意**：过度优化可能影响预测精度，建议在速度和精度之间找到平衡点。

## 注意事项

1. ML 预测现在只需 **15-30 秒**（优化后）
2. 预测期间不要关闭浏览器标签页
3. 如遇错误，可查看服务器终端日志
4. 建议在非高峰时段运行 ML 预测
5. 优化后速度提升约 50%，精度影响很小
